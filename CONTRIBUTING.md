# 贡献指南

感谢您对 3ds Max 插件开发知识库的兴趣！我们欢迎所有形式的贡献。

## 如何贡献

### 1. 报告问题

如果您发现文档中的错误或有改进建议：

1. 检查是否已有相关 Issue
2. 创建新 Issue，清楚描述问题或建议
3. 如果可能，提供复现步骤或示例代码

### 2. 提交代码

#### 准备工作

1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启 Pull Request

#### 代码规范

**MaxScript:**
- 使用有意义的函数和变量名
- 添加适当的注释
- 使用 `try-catch` 处理异常

**Python:**
- 遵循 PEP 8 规范
- 使用类型提示
- 编写 docstring

**C++:**
- 遵循 3ds Max SDK 编码规范
- 使用现代 C++ 特性（C++14/17）
- 添加头文件保护

### 3. 文档贡献

#### 文档格式

- 使用 Markdown 格式
- 包含清晰的标题结构
- 提供代码示例
- 使用中文编写（可附英文对照）

#### 目录结构

```
docs/
├── maxscript/        # MaxScript 相关文档
├── python/           # Python 相关文档
├── cpp-sdk/          # C++ SDK 相关文档
├── api-reference/    # API 参考
└── best-practices/   # 最佳实践

examples/
├── maxscript/        # MaxScript 示例
├── python/           # Python 示例
└── cpp-sdk/          # C++ SDK 示例
```

### 4. 代码示例贡献

提交代码示例时，请确保：

1. 代码可以直接运行
2. 包含清晰的注释说明
3. 涵盖常见用例
4. 遵循最佳实践

#### 示例文件模板

**MaxScript (`.ms`):**
```maxscript
-- ========================================
-- 标题
-- ========================================
-- 描述：功能说明
-- 作者：您的名字
-- 日期：YYYY-MM-DD

-- 代码示例...
```

**Python (`.py`):**
```python
"""
标题
========================================
描述：功能说明
作者：您的名字
日期：YYYY-MM-DD
"""

# 代码示例...
```

**C++ (`.cpp`):**
```cpp
/**
 * 标题
 * ========================================
 * 描述：功能说明
 * 作者：您的名字
 * 日期：YYYY-MM-DD
 */

// 代码示例...
```

## 审核流程

1. 提交 Pull Request
2. 等待代码审核
3. 根据反馈进行修改
4. 合并到主分支

## 社区准则

- 尊重所有贡献者
- 提供建设性反馈
- 保持专业和友好的交流
- 遵守开源许可证

## 联系方式

如有任何问题，请通过 Issue 联系我们。

感谢您的贡献！
